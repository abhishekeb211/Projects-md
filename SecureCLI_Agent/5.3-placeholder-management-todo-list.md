## Placeholder Extraction Process

### Extraction Methodology

Placeholders were extracted from:

1. **Main LaTeX Document (`aegiscli-paper.tex`):**
   - Abstract TODO: `\todo{Write abstract (150-200 words)...}`
   - Appendix TODOs: `\todo{Include SARIF v2.1.0 subset schema JSON}` (4 appendices)
   - Artifact DOI TODO: `\todo{Zenodo DOI}`

2. **Section Files (`sections/*.tex`):**
   - Experimental results: `\result{E1}{TBD}` (if present)
   - Content TODOs: Section-specific action items

3. **Phase 4 Markdown Files (`4.*.md`):**
   - Evaluation setup placeholders: `\todo{TBD - Fill with actual results}`
   - Figure generation: `\todo{TBD - Generate histogram figure}`
   - IRB protocol: `[TBD—IRB submission pending]`

### Placeholder Types Identified

1. **Result Placeholders:** Experimental data requiring P3 results (Tables 3-5, Figure 4)
2. **Content Placeholders:** Sections requiring LaTeX conversion or content additions
3. **Appendix Placeholders:** Appendices requiring schema, policy, prompt, protocol content
4. **Reference Placeholders:** Bibliography, citation verification, figure references
5. **Validation Placeholders:** Data validation tasks (demographics, benchmark repos)

## Action Tracking Framework

### Tracking Mechanism

TODO tasks are tracked via markdown table in `TODO-master.md` with columns:

- **ID:** Unique identifier (T01-T20)
- **Location:** Section/appendix reference (e.g., "Sec 6.1", "Appendix A")
- **Description:** Actionable description
- **Owner:** Assigned team member (Researcher1, Researcher2, Engineer1)
- **Deadline:** Week target (W48-W52)
- **Priority:** P0/P1/P2 classification

### Completion Workflow

1. **Task Assignment:** Owner assigned based on expertise (Researcher1: results/validation, Researcher2: qualitative content, Engineer1: technical artifacts)
2. **Deadline Setting:** Deadlines set based on submission timeline (W48: early, W50: results, W52: final)
3. **Priority Classification:** P0 blocks submission, P1 required for completeness, P2 polishing
4. **Progress Tracking:** Status updates (pending/in-progress/completed) via TODO list

### Quality Gates

- **P0 Threshold:** P0 tasks must be <50% of total (7/19 = 37% ✅)
- **Owner Assignment:** All tasks have assigned owners ✅
- **Deadline Coverage:** All tasks have deadline targets ✅
- **Priority Distribution:** Balanced P0/P1/P2 distribution ✅

## Integration with LaTeX Document

Placeholders are integrated into LaTeX via:

1. **`\todo{}` Command:** `\todo{description}` (defined in `macros.tex`)
   - Renders as red text: `[TODO: description]`
   - Visible in draft, removed before submission

2. **`\result{}` Command:** `\result{exp}{TBD}` (defined in `macros.tex`)
   - Format: `E1: TBD` (experiment label and placeholder)
   - Replaced with actual results when available

3. **Placeholder Preservation:** Placeholders preserved during markdown→LaTeX conversion for action tracking
